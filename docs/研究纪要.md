# 研究纪要 ——「自进化 skill 其实是低秩的」

> 现阶段工作的**中文统一纪要**(对应英文 `docs/RESEARCH_NARRATIVE.md`)。
> 所有标 ✅ 的结论均在本地 ghc 代理 + gpt-5.5 上实测,**确定性 EM** 评分(无模型裁判)。
> 代码在 `litaohz/SkillOpt` 分支 `research/a3-skill-retrieval`,已同步 `gim-home/skillopt` main。

---

## 一句话结论

**SkillOpt 这类「自进化 skill」优化的有效维度极低**:真正学到的东西既集中在**时间**上(只有头一两个被接受的编辑 / 几个 epoch),又集中在**内容**上(skill 里只有几句话),其余的算力都在产出**因果价值≈0、甚至有害**的冗余文本。我们用**因果归因**既**诊断**了这点,又给出**更省的优化方向**。

> 这两种稀疏是**同一现象的两个投影**:后面那些 epoch 没用,正是因为它们新增的文本因果价值≈0(与已有内容冗余,或有害)。

## 二、工具:因果归因(干预测量,不是相关)

把「加了这段分数涨了」(相关)换成 **do 算子**测量(消融某单元/子集后重测)。

- **一阶**(`scripts/skill_attribution.py`):每个单元的 LOO Δ(不可或缺度)、add-one Δ(独立价值)。能看出稀疏,但会被**冗余**(替代品互相掩盖)和**协同**(只在搭档在场才有用)骗。
- **组合 / 交互感知**(`scripts/skill_combo.py`,**主线 D**):二阶交互 `I(i,j)=s(full)−s(\i)−s(\j)+s(\{i,j})`(I<0 替代/冗余,I>0 协同),以及预算化 Monte-Carlo Shapley。给出**真正的最小充分子集**。所有子集评测按 skill 哈希**磁盘缓存、跨方法复用**(省评测次数)。

> 省钱原则:贵的是评测**次数**,不是题量。交互是二阶差分,需要题量保统计功效;省钱靠**定向对 + 采样 + 缓存复用**。

## 三、关键实测结果(OfficeQA,172 测试)

### 1. gate 选中的「最佳 skill」:19 单元里只有 1 句有用 ✅
- full(19 单元)= 0.692,empty(无 skill)= 0.547。
- **#17 输出格式句**:LOO **+0.145**(删它直接掉回 0.547=无 skill),add-one **+0.174**;
  **单加这一句 = 0.721,反超 19 单元全量 0.692**。
- 其余 18 条 ≈0 或负;长算术规则单独加 −0.19 / −0.30(有害)。

### 2. 因果剪枝 ✅
- 留含 #17 的 **4 条(470 字符)= 0.715**,**追平整个 16-epoch 训练**(best 0.733 / final 0.715),体积小 5×。

### 3. 交互矩阵:零协同 ✅
- 对 {10,12,13,17} 定向 pairwise:**所有 I(i,j)≤0**,无一对是协同。
- 算术规则彼此是替代品、且与 #17 冗余 → **「删了不亏」不是被冗余骗的,是真死重;剪枝不丢协同**。

### 4. 桥实验:时间稀疏↔内容稀疏焊死 ✅
- 膨胀终版(20,238 字符,含 18k 强制注入的 slow_update)= 0.709;
- **剥掉整个 slow_update 块(2,488 字符)= 0.715**(EM 反升 +0.006,体积小 8×)。
- 即:**28 个无效步骤 force-accept 进来的全部 16k 文本,因果价值 = 0**。
- 整个 7.6h / 1.06 亿 token 的训练成果,**可被 ~4 句话复现甚至超过**。

## 四、修复方向:因果指导的全局优化

因为价值稀疏、且优化器是「盲的」(自报 meta-skill + 强制接受 + 贪心 gate):
1. **按实测价值剪枝** → 压缩;多目标 **Pareto(EM, token)**,用每单元的(价值, 成本)做背包剪枝。治膨胀。
2. **指导优化器**:把实测的每单元价值注回 meta-skill / analyst prompt(`skillopt/optimizer/causal_hints.py` 已实现注入块:Protect / Remove / Redundant),让编辑按**实测价值**而非自报取舍。局部噪声 SGD → 全局 credit assignment。
3. **harness 级全局视角**:对**所有可编辑面**(模板 Rules、skill、工具说明、优化器 prompt)联合归因,决定「每份价值该放在哪一层」。

## 五、贡献(paper 骨架)

1. **方法**:harness 级因果归因框架(一阶 → 组合 / 交互感知)。
2. **实证**:agentic skill 学习的**双重稀疏 + 机制**(强制更新、贪心早锁、冗余)。
3. **构造**:因果指导、成本感知、全局优化器,**同等精度、零头的 token/epoch**。

**对外一句话**:*"Self-evolving" skills are low-rank —— skill 优化集中在几次编辑、几句话上,其余是浪费;因果归因既证明又修复了它。*
